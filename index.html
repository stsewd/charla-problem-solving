Aclaración

Después de esta charla no vas a ganar una competencia. No es el objetivo. Pero si vas a estar más preparado.

Lo que debes conocer

Hay varios temas que generalmente se tratan en estas competencias.

Big O notation

Usada para clasificar algoritmos de acuerdo a su tiempo de ejecución o uso de recursos.

O(1) O(n2) O(n!)
O(n) O(n3)
O(n * log(n)) O(nx)

Nos da una medida de que tan mal nuestro algoritmo va a ejecutar con cierta entrada de datos. Siempre nos da el peor caso. ¿Alguien recuerda esto? Nos sirve mucho en este tipo de competencias porque..., y en le vida real.

Algoritmos de ordenamiento y búsqueda

  • Bubble sort, selection sort, insertion sort, merge sort, quick sort, counting sort.
  • Linear search, binary search.

Aprende como funcionan, pero tampoco debes reinventarlos.

Hacer énfasis en counting sort, probablemente nunca les enseñaron

Usar las funciones de ordenamiento que el lenguaje nos provee

Ejemplo de sorted y .sort en python

Estructuras de datos

  • Arrays, stacks, queues.
  • Sets (teoría de conjuntos)
  • Árboles y grafos (teoría de grafos)
Para cada problema es necesario usar las estructuras de datos adecuadas y sus operaciones. Nos pueden ahorrar memoria, y tiempo de ejecución. Mostrar ejemplos en python (pop, append), sets más rápido que lists para buscar. Grafos, matrix de adyacencia (evitar hacer un objecto), dirigidos, no dirigidos.

Teoría de números

  • Números primos
  • Greatest common divisor (GCD) (algoritmo de euclides)
  • Aritmética modular
Señalar la importancia de los números primos, y aritmética modular. Mostrat algunos ejemplos en python (math.gcd). El alg de euclides es uno de los alg más antiguos.

Manipulación de bits

  • NOT (~)
  • AND (&)
  • OR (|)
  • XOR (^)
  • Left shift (<<)
  • Right shift (>>)
Mostrat ejemplos en python 0b12, énfasis en XOR. Left shift n * 2**k, right shift n/2*k

Operaciones con strings

  • Substitución
  • String matching
  • Expresiones regulares
Cuantos caracteres debes cambiar o eliminar para que sea igual que otro string. Hay algoritmos de búsqueda de substrings. Tratar de no usarlas. Tener cuidado del uso de memoria con strings.

Optimizar

  • Greedy algorithm
  • Dynamic programming
Cuidado con el nombre. Maneras de implementar un algoritmo, no son un algoritmo o un paradigma de programación. Ejemplos: más tareas en el menor tiempo posible (no siempre un greedy resuelve el problema). Factorial, memorización.

Resolviendo problemas

Resolviendo problemas

Primero resuelve el problema, luego programa.

Si empiezas a programar primero, no vas a tener idea de lo que estás programando. Usa lápiz y papel.

Divide el problema en problemas pequeños.

Pequeño problemas son más fáciles de entender y resolver.

Presta atención a los límites y tipo de los valores. Prueba tu algoritmo con valores grandes.

Muchas veces tu algoritmo va a fallar, y entiendes porqué. O escribes código muy defensivo cuando no es necesario.

Primero resulve el problema de la manera más tonta, optimiza donde sea necesario.

Antes de complicarte con soluciones complejas, hazlo simple. Incluso entiendes mejor el problema. Hazlo con fuerza bruta, puede que esa sea la solución. También se aplica en software YAGNI.

Explica a alguien más el problema.

También aplica cuando arreglas un bug. Explicandole a alguien más el problema te puede traer otras ideas. Pragmatic programmer, ruber duck debugging.

Tips

  • Google es tu amigo
  • Usa lo que te sea más cómodo
  • Usa tu editor o IDE favorito
  • Escoge tu lenguaje de programación
  • Escribe código aceptablemente limpio
Hay muchas cosas por conocer! Pero por suerte no debemos memorizar cada algoritmo. Pero si entenderlo y saber que existe para solucionar un tipo de problema. Escribe codigo limpio, así otros miembros de tu equipo de pueden ayudar.

Lenguaje de programación

  • Librería estándar muy rica
  • Conoce sus estructuras de datos
  • Conoce sus peculiaridades
  • Conoce como usar su debugger
Ejemplos: python, c++, java (no brainfuck). No tienes porque escribir todos los algoritmos desde cero.
